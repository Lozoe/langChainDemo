🔍 开发规范检查器 (MCP Server) 分析报告

您提供的 Python 代码使用 FastMCP 框架创建了一个名为 "开发规范检查器" 的 MCP (Model Context Protocol) 应用程序接口（Server）。目前，通过 MCP 协议，大语言模型或各种代理可以方便地调用服务器对外暴露的接口工具。这个程序就是一套服务于代码质量检查和规范管控的工具集合。

🛠️ 包含的工具模块详细说明

1. check_code_style (检查代码风格规范)

功能：接收一个目录路径，提取并读取其下所有的 *.py 文件内容进行风格审计。

校验规则：

  • 单行最大长度是否超过 88 个字符 (借鉴了 Python Black 风格规范器)。
  • 文件是否存在模块级的 docstring (是否包含 """ 或 ''')。
  • 粗略检查变量命名是否不规范（利用正则找出形如全大写但被用作局部变量的名字，以判定是否违背应该下划线全小写的原则）。
2. search_todo_comments (搜索TODO相关注释)

功能：扫描项目中常见源代码文件格式 (如 .py, .js, .java, .cpp 等)，利用正则全词忽略大小写匹配的方式找出技术债或待补全记录。

关注的标记： TODO:, FIXME:, XXX:, HACK:

3. validate_project_structure (验证项目结构)

功能：探测执行环境下的文件和目录存在情况，判断是否符合某语言主流架构的"最佳实践"并生成推荐提示。

判定模式（当前支持配置）：

  • Python：检查 requirements.txt, setup.py 等和 src, tests 文件夹等是否存在。
  • JavaScript：检查 package.json, vite/webpack.config.js 和 dist, public 等。
4. get_naming_conventions (获取命名规范)

功能：一个纯数据形式的查询接口，向大模型下发特定语言的标准规范定义文本。

涵盖内容：Python, JavaScript, Java, Go 四种语言各层级（变量、函数、类、包）应该使用的是 snake_case, camelCase 还是 PascalCase 形式。

5. check_import_organization (检查导入语句组织)

功能：对输入的一份 Python 文件专门做导入依赖规范度巡查。

判断逻辑：系统内写死列举了主流的 stdlib_modules 标准库名列表。当扫描到 import 时，提取第一个包层级做比对，借此分划归类为：标准库导入、第三方库导入（从而可以指导开发者分段落排版）。

💡 整体评价与用途

该脚本是一个非常有实用价值的代码审查和辅助插件。当它被大模型连接使用时，模型就能拥有了“实时探测本地文件代码坏味道”、“寻找代办”的超能力，成为你的项目质量哨兵或“自动化代码格式审查助手”。

脚本末尾做了很好的启动设计，当使用 mcp dev 执行时，会使用底层 server 原生运行；而如果手动命令执行则使用 FastMCP 自身的 run 循环运行。
